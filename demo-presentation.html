02 / 07
The Problem
AI agents read websites every day. Publishers have no way to monetize this traffic.
"Google AdSense doesn't work for AI agents. There's no way to detect, price, or monetize agent-generated traffic."
  • Agents scrape content without paying publishers
  • No micro-payment infrastructure exists for machine-to-machine HTTP requests
  • Advertisers have no mechanism to reach AI agents mid-query
  • Existing paywalls are friction-heavy and designed for humans, not APIs
AgentSense solves all three.
03 / 07
The Solution
A monetization layer for content that AI agents read.
For Publishers
Earn USDC per query
Paste one middleware function onto any endpoint. Every AI query earns you USDC — automatically, on-chain, with no contracts to manage.
For Advertisers
Bid for agent attention
Fund a session with keywords and a bid. When an agent queries matching content, your sponsored result is injected into the response.
For Agents
Pay transparently
$0.01 per query, paid in USDC via Stellar. Single HTTP request-response cycle. No gas fees. No wallet drainage.
The Platform
Runs the auction
AgentSense takes 0.7% from ad budgets. Publishers earn 0.7% of advertiser bids per query. The platform sets prices.

07 / 07
Why Stellar + x402
The only blockchain with native HTTP payment primitives for machines.
x402 Protocol
HTTP-native
x402 embeds payment into HTTP headers. No blockchain RPC calls, no custom wallets. The agent's SDK handles everything in one request-response cycle.
Soroban Auth
Zero gas for clients
Soroban authorization entries let wallets authorize transactions without paying gas. The OZ Channels facilitator sponsors fees — agents just sign.
OZ Channels
Fee sponsorship
OpenZeppelin's Channels service verifies payments and settles on-chain. Publishers get direct USDC transfers. No intermediary risk.
"Stellar's x402 + Soroban auth entries are the only production-ready way to do per-request on-chain micro-payments in a standard HTTP client."
